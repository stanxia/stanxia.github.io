spark关于parquet的优化

发表于 2017-11-01 | 分类于 spark | | 阅读次数:

parquet是一种列式存储。可以提供面向列的存储和查询。

Parquet的优势

在sparkSQL程序中使用parquet格式存储文件，在存储空间和查询性能方面都有很高的效率。

存储方面

因为是面向列的存储，同一列的类型相同，因而在存储的过程中可以使用更高效的压缩方案，可以节省大量的存储空间。

查询方面

在执行查询任务时，只会扫描需要的列，而不是全部，高度灵活性使查询变得非常高效。

实例测试
